This dcoument contains instructions for starting and stoping the Servlet Engine.

Unix

During the installation a script was copied to /etc/init.d/. This script can be used to start and stop the server. The following commands allow you to start, stop and restart the server:

  • To start:
    /etc/init.d/orangefood-servletengine start
  • To stop:
    /etc/init.d/orangefood-servletengine stop
  • To restart:
    /etc/init.d/orangefood-servletengine restart

To make the server start when the system is turned on or restarted either use the chkconfig program or create links to the init script in the approprate runlevel directories.

Windows

The installation instructions describe how to install the server as a Windows Service. Once the service has been configured you can start, stop and restart the server using the service control methods provided by windows. A windows service can also be configured to start when the system is turned on or restarted.

Service Control Panel

Widows provides a graphics interface for controling services. To use this method you must open the Control Panel and then open the Services item (in newer versions of Windows this is located in the Administrative Tools folder). The service is installed as "Orangefood.com Servlet Engine".

Command Line

Windows services can also be controled on the commanf line. Use the following commands to stop or start the server:

  • To start:
    net start "Orangefood.com Servlet Engine"
  • To stop:
    net stop "Orangefood.com Servlet Engine"

Manual

You can also start or stop the server manually by invoking the Java Virtual Machine directly. The server is started when the JVM is passed the name of the main class (com.orangefood.se.Server). The server then listens for a shutdown command with the approprate password. A class is provided to send the shutdown command, com.orangefood.se.StopServer. These instructions assume that the default port (9009) and password (shutdownpass) are being used, see the configure document for instructions on how to change the default port and password. To start or stop the server, change to the directory into which the orangefood-servletengine.jar was installed and use the following commands:

  • To start:
    java -jar orangefood-servletengine.jar
  • To stop:
    java -cp ornagefood-servletengine.jar com.orangefood.se.StopServer shutdownpass 9009

Options

When the server is started additional configuration locations can be passed in on the command line. The configuration locations are specified as URLs. The server will search for configuration files (e.g. server.xml, web.xml) starting with the first specified location until the file is found. The default configuration files are located in the config/ directory within the .jar file. Examples:

  • To use the directory /etc/servlet-config/ as a configuration directory:
    java -jar orangefood-servletengine.jar file:/etc/servlet-config/
  • To use the directory /config on a webserver hosted on servlet.orangefood.com:
    java -jar orangefood-servletengine.jar http://servlet.orangefood.com/config/
  • Multiple locations can be specified and will be searched in order. To combine the two locations from above, searching first in the directory then the web server:
    java -jar orangefood-servletengine.jar file:/etc/servlet-config/ http://servlet.orangefood.com/config/
